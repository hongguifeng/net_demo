# 第一章：以太网帧解析与构造

## 原理

以太网帧是数据链路层 (L2) 的基本传输单元。理解帧结构是理解整个协议栈的基础。

### 以太网帧格式 (Ethernet II)

```
┌──────────────────────────────────────────────────────────────────┐
│ 前导码(8B) │ 目的MAC(6B) │ 源MAC(6B) │ 类型(2B) │ 数据(46-1500B) │ FCS(4B) │
└──────────────────────────────────────────────────────────────────┘
```

- **目的 MAC 地址** (6 字节)：接收方的硬件地址
- **源 MAC 地址** (6 字节)：发送方的硬件地址
- **类型/长度** (2 字节)：标识上层协议
  - `0x0800` = IPv4
  - `0x0806` = ARP
  - `0x86DD` = IPv6
- **数据** (46-1500 字节)：有效载荷
- **FCS** (4 字节)：帧校验序列 (CRC32)

### 关键概念

1. **MTU (Maximum Transmission Unit)**：以太网默认 MTU 为 1500 字节
2. **广播地址**：`ff:ff:ff:ff:ff:ff`，所有同一广播域的主机都会接收
3. **混杂模式**：网卡接收所有经过的帧，而非仅目标是自己的帧

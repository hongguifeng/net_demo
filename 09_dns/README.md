# 第九章：DNS 协议实现

## 原理

DNS (Domain Name System) 将域名映射为 IP 地址，是互联网最关键的基础协议之一。

### DNS 查询流程

```
浏览器输入 www.example.com
        │
        ▼
┌─────────────────┐     ┌──────────────────┐
│ 本地 DNS 缓存    │────→│ /etc/hosts       │
└────────┬────────┘     └──────────────────┘
         │ (缓存未命中)
         ▼
┌─────────────────┐     ┌──────────────────┐
│ 递归解析器       │────→│ 如: 8.8.8.8      │
│ (ISP/公共 DNS)  │     │     1.1.1.1      │
└────────┬────────┘     └──────────────────┘
         │ (递归查询)
         ▼
┌─────────────────┐
│ 根域名服务器 (.) │ → 返回 .com 的 NS
└────────┬────────┘
         ▼
┌─────────────────┐
│ .com TLD 服务器  │ → 返回 example.com 的 NS
└────────┬────────┘
         ▼
┌─────────────────┐
│ example.com     │ → 返回 www.example.com 的 A 记录
│ 权威服务器       │
└─────────────────┘
```

### DNS 消息格式

```
+---------------------+
|        Header       |  (12 bytes 固定)
+---------------------+
|       Question      |  查询名称和类型
+---------------------+
|        Answer       |  响应资源记录
+---------------------+
|      Authority      |  权威服务器记录
+---------------------+
|      Additional     |  附加记录
+---------------------+

Header:
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                      ID                         |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|QR|   Opcode  |AA|TC|RD|RA|   Z    |   RCODE   |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    QDCOUNT                      |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ANCOUNT                      |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    NSCOUNT                      |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
|                    ARCOUNT                      |
+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+--+
```

### 记录类型

| 类型 | 值 | 说明 |
|------|-----|------|
| A | 1 | IPv4 地址 |
| AAAA | 28 | IPv6 地址 |
| CNAME | 5 | 别名 |
| MX | 15 | 邮件服务器 |
| NS | 2 | 域名服务器 |
| TXT | 16 | 文本记录 |
| SOA | 6 | 起始授权 |
| PTR | 12 | 反向解析 |

### DNS 安全

- **DNS 劫持**: 篡改 DNS 响应，将域名指向恶意 IP
- **DNS 污染**: 向缓存注入虚假记录
- **DNSSEC**: 通过数字签名验证 DNS 响应真实性
- **DoH/DoT**: DNS over HTTPS/TLS，加密 DNS 查询

## 示例代码

| 文件 | 说明 |
|------|------|
| `dns_resolver.py` | 从零实现 DNS 解析器 |

## 运行

```bash
python3 dns_resolver.py example.com
python3 dns_server.py &
dig @127.0.0.1 -p 5353 example.com
```

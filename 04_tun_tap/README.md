# 第四章：TUN/TAP 虚拟网络接口

## 原理

TUN/TAP 是 Linux 内核提供的虚拟网络设备，是实现用户态网络协议栈、VPN、隧道的基础。

### TUN vs TAP

```
┌─────────────────────────────────────────────────────────┐
│                    用户空间应用                           │
│                  (VPN, 隧道程序)                         │
└─────────────────────┬───────────────────────────────────┘
                      │ read()/write() (文件描述符)
                      │
┌─────────────────────┴───────────────────────────────────┐
│                  /dev/net/tun                            │
│           ┌─────────────┬──────────────┐                │
│           │  TUN 设备    │  TAP 设备     │                │
│           │  (L3, IP包)  │  (L2, 以太帧) │                │
│           └──────┬──────┴───────┬──────┘                │
│                  │              │                         │
│           内核网络栈 (路由、转发等)                        │
└─────────────────────────────────────────────────────────┘
```

| 特性 | TUN | TAP |
|------|-----|-----|
| 层级 | L3 (网络层) | L2 (数据链路层) |
| 数据格式 | IP 数据包 | 以太网帧 |
| 用途 | VPN (如 WireGuard) | 虚拟交换机 (如 QEMU) |
| 头部 | 无以太网头 | 包含以太网头 |

### 工作流程

```
应用程序 write(tun_fd, ip_packet)
    ↓
内核收到 IP 包，查路由表
    ↓
根据路由决定转发/本地处理
    ↓
如果目标匹配 tun 接口路由，包被送到 tun_fd
    ↓
应用程序 read(tun_fd) 取出包
```

### 应用场景

1. **VPN 客户端**：将加密的 IP 包写入 TUN，内核路由到正确的网络
2. **容器网络**：Docker/K8s 使用 veth+bridge+TUN 构建容器网络
3. **网络模拟**：用户态协议栈测试
4. **流量劫持**：透明代理

## 示例代码

| 文件 | 说明 |
|------|------|
| `tun_device.py` | 创建 TUN 设备并收发 IP 包 |
| `tap_bridge.py` | 创建 TAP 设备模拟二层桥接 |
| `tun_echo.py` | TUN 回显 - ICMP ping 响应器 |

## 运行

```bash
sudo python3 tun_device.py    # 创建 TUN 并 ping
sudo python3 tun_echo.py      # TUN ICMP 响应器
sudo python3 tap_bridge.py    # TAP 桥接演示
```
